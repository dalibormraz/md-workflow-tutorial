# Sourcecheck — aplikace

## Co to je

Rozšíření do prohlížeče (a později appka), která k označenému tvrzení najde 2–3 důvěryhodné zdroje k ověření.

## Pro koho

  • Lidé 25–55 let
  • Čtou online zprávy a sociální sítě
  • Chtějí si ověřit, než to sdílí

## Co uživatel odnese

Do 10 sekund 2–3 ověřitelné zdroje — nebo varování, že tomu nic seriózního nepotvrzuje.

## Co to NENÍ

  • Není to fact-checking redakce s lidskými verdikty
  • Není to nástroj pro novináře — je to pro běžné lidi

## Odlišení

Rychlost a kontext. Funguje přímo tam, kde čteš. Žádné přepínání tabů.

Verzuj. Pořád.

# Know-how — Sourcecheck

## Důvěryhodné zdroje v ČR

  • Tier 1 (vysoká důvěra): ČT, ČRo, Seznam Zprávy, iROZHLAS, investigace.cz
  • Tier 2 (s opatrností): mainstream deníky — kontext rubriky rozhoduje
  • Aggregátory (Wikipedie, primární studie) = startovní bod, ne finální verdikt

## Co uživatele 25–55 nesnáší

  • Když musí kliknout 3× pro výsledek
  • Když mu appka říká „pravda / lež" bez vysvětlení proč
  • Dlouhé úvody — do 10 sekund musí vidět zdroje

## Frameworky, které fungují

  • Tvrzení → Zdroj → Kontext — struktura každého výsledku
  • Pravidlo tří zdrojů — vždy 2–3, nikdy jen jeden

## Moje červené čáry

  • Žádné politické soudy ani „pravda / lež" verdikty
  • Žádný tracking ani prodej dat uživatelů

# Odpovědi — Sourcecheck

## Q1: Jak má vypadat výsledek, aby mu lidi věřili?

Vždy 3 zdroje, každý s jednou větou kontextu, proč ho ukazujeme. Žádný „verdikt" nahoře — to lidi okamžitě uzavře. Nech závěr na uživateli.

## Q2: Jak řešit, když není co najít?

Nejdůležitější věc celé aplikace. „Nenašli jsme nic seriózního" + 2–3 tipy, kde hledat dál. Nikdy si nic nevymýšlej.

## Q3: Co by neměla být první feature?

Ne komplet fact-check pipeline. Začni něčím malým — třeba „označ větu → vrátí 3 zdroje z důvěryhodných médií".

# Sourcecheck — koncept 1

## Formát

Rozšíření pro Chrome a Firefox. Framework Tvrzení → Zdroj → Kontext. Výsledek do 10 sekund.

## Uživatelský flow

  • 0–2s: uživatel označí větu na stránce
  • 2–10s: appka najde 2–3 zdroje
  • 10s+: panel s odkazy + jednovětý kontext u každého
  • Fallback: „nic seriózního jsme nenašli" + tipy kam dál

## MVP — první 3 funkce

  • Označ větu → 3 zdroje z Tier 1 médií
  • Indikace, když zdroj neexistuje
  • Uložení na později (pro mobil)
  • /* ... */

## Tým a workflow

3 lidi: produkťák + frontend + ML/data inženýr. MVP = 6–8 týdnů práce.

# Specifikace — Sourcecheck, MVP

## 1. Technologie

  • Frontend: rozšíření pro Chrome + Firefox (Manifest V3, TypeScript)
  • Backend: Node.js + Fastify, hosting Vercel / Fly.io
  • Vyhledávání zdrojů: vlastní index Tier 1 médií (RSS + sitemap crawler)
  • LLM pro extrakci tvrzení: Claude Haiku přes API

## 2. Klíčové komponenty

  • Content script: označení textu na stránce → odeslání do API
  • Service: extrakce klíčového tvrzení (LLM) + dotaz do indexu
  • Ranking: relevance + důvěryhodnost zdroje + čerstvost článku

## 3. Časový plán

  • Týden 1–2: crawler + index Tier 1 médií
  • Týden 3–4: backend API + ranking
  • Týden 5–6: Chrome rozšíření + UX panel
  • Týden 7–8: uzavřená beta s 20 testery

## 4. Rozpočet — MVP

  • Tým (3 lidé × 8 týdnů): 720 000 Kč
  • Infrastruktura (hosting + API kredity): 20 000 Kč
  • Designer na výplatu (UX panel, ikony): 40 000 Kč
  • Rezerva 15 %: 120 000 Kč
  • Celkem MVP: ~900 000 Kč

## 5. Milníky

  • M1 (T+2 týdny): index Tier 1 zdrojů funguje
  • M2 (T+4 týdny): API vrací 3 zdroje na tvrzení
  • M3 (T+8 týdnů): beta hotová, 20 lidí používá
